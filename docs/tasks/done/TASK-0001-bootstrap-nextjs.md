# TASK-0001 — Bootstrap the Next.js foundation (P0 / Milestone M1)

## Context

The repo is pre-implementation — only `LICENSE`, `docs/`, and content tooling
(`lib/content/sprites.ts`, `public/sprites/`, `tools/sprites/`) exist. Before any section
can be built, the app shell, design system, and content-model types must exist. This is the
**critical-path start** of the build per the planning package.

Read first: [[../../setup/getting-started]] (Bootstrap), [[../../architecture/overview]],
[[../../architecture/data-flow]], [[../../planning/06-execution]] (Phase P0 / tasks
D-01..D-08), and the repo `CLAUDE.md` (pnpm-only + docs rules).

## Goal

A Next.js 15 + React 19 + TypeScript + Tailwind app that builds, lints, and deploys a blank
but styled shell — with design tokens, fonts, Lenis smooth scroll, and all content-model
types in place — ready for section work to begin.

## Scope

**In scope**
- Bootstrap Next.js 15 (App Router) + React 19 + TypeScript via **pnpm** (D-01).
- ESLint + Prettier + `tsconfig` + path aliases; commit `pnpm-lock.yaml` (D-02).
- Tailwind config + `globals.css` with the design tokens from
  [[../../planning/03-storyboard-motion-visual#8-2-design-tokens]] as CSS vars (D-03).
- Fonts via `next/font`, subset, `display: swap` (D-04).
- Install + wire **Lenis** smooth scroll as a root client island (D-05).
- Define content-model TypeScript types in `lib/content/`: `hero`, `loop`, `modes`,
  `family`, plus the already-specified `features`, `themes`, `faq`, `screenshots` (D-06).
  Keep the **auto-generated** `lib/content/sprites.ts` untouched.
- `prefers-reduced-motion` + low-power detection utility (D-08).
- Root `app/layout.tsx` shell (metadata, fonts, Lenis provider) + empty `app/page.tsx`.

**Out of scope**
- Any actual section UI (Hero, Reveal, etc.) — those are later tasks.
- The waitlist API route + provider (TASK to follow; needs an ADR).
- i18n locales, SEO schema, depth pages, blog — later phases (P3/P4).
- Editing `lib/content/sprites.ts` by hand (it is generated by `tools/sprites/`).

## Relevant Files

- `package.json`, `pnpm-lock.yaml`, `next.config.js`, `tsconfig.json` — create (bootstrap).
- `tailwind.config.ts` — create; extend theme with planning tokens.
- `app/layout.tsx`, `app/page.tsx`, `app/globals.css` — create.
- `lib/content/hero.ts`, `lib/content/loop.ts`, `lib/content/modes.ts`,
  `lib/content/family.ts`, `lib/content/features.ts`, `lib/content/themes.ts`,
  `lib/content/faq.ts`, `lib/content/screenshots.ts` — create typed models
  (shapes in [[../../architecture/data-flow]] + [[../../planning/02-architecture#5-1-new-content-models-required]]).
- `lib/content/sprites.ts` — **read only**, do not edit (generated).
- `lib/motion.ts` (or similar) — reduced-motion / low-power helper.

## Technical Notes

- **pnpm only** (repo `CLAUDE.md`): `pnpm add` / `pnpm add -D` / `pnpm dlx`. Never npm/yarn.
- Stack is fixed: Next.js 15, React 19, Tailwind, GSAP ScrollTrigger, Framer Motion, Lenis,
  React Three Fiber, Drei — but only **Lenis + Tailwind** are wired in this task; GSAP/Framer/
  R3F/Drei are installed when their first consuming section lands (keep the initial bundle lean).
- Default to **Server Components**; the Lenis provider is the only client island here.
- Design tokens are the source of truth in [[../../planning/03-storyboard-motion-visual#8-2-design-tokens]];
  do not invent new colors.
- See [[../../decisions/ADR-0001-web-stack]] for the stack rationale.
